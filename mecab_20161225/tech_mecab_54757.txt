supervisor這個職務的描述並不算很多，也看得出來很多新鮮人對"帶線"到底在做什麼感到很好奇，有鑑於此，小弟忍不住想和各位分享一下近兩年來的帶線心得~------------------------------------------------------------------------------先說一下自己，私立工工--&gt;國立工管/工工碩，算正規 IE 出身，這工作並不是第一份工作，之前曾在社會打混過2~3年，也算是嘗試過一些不同性質的工作。其實帶線要做的事很多、很雜，工廠裡大大小小的事你都會莫名被捲入(很多事要算你的)，對一個沒有工作經驗的新人來說其實是蠻吃力的，這也難怪陣亡率很高很高...但是如果能撐過去，會發現學到很多，也會發現...產線的問題還真的是多到靠北在我任職的晶圓代工廠裡，對產線課長的要求還不少，我還是新人時曾問過我學長，問他這幾年的心得，那時他只淡淡的回我一句："不是人幹的"   XD雖然很多人認為入門門檻低，只要耐操就能做，但就我本身的經驗來看這樣想就太簡單了，個人觀感是覺得帶線要能做得ok大致有以下幾條要求(只有體力好，大概會每天被定)1. 要對生產有Sense，至少要了解一些生產管理的內涵，尤其在現場管理、排程這一塊2. 體力要ok，能承受輪班、10幾小時的轟炸 (工程師都一樣，輪班星人的必備要求)3. 抗壓性要強，尤其是生產會議...被定的時候要能把自己當空氣，委屈？算個屁XD4. 溝通能力要好，要有邏輯條理，對上對下都要有一套 (嘴砲技能要點滿，但也要有料)5. 要有基本的資料處理能力，尤其是在有限時間內要能夠整理出重點，下達適當的指令6. 反應要快，尤其產線偶爾會發生一些緊急事件，要能迅速作決斷，且要有一些指揮能力簡單來說，產線的 supervisor 就是一個入門版的主管，直接就要掌控產線，同時也是工程師，要能針對產線的現況運用 IE 手法進行分析、改善。常常有人不知道產線課長/主任到底在幹嘛？平常在做什麼？其實大致上如下：1. 隨時監控產線，針對誤差的部分下達指令做調整，維持產線的正常運作，包括進行動態的排程、派工，以保持 WIP 結構正常以及 Line-balance，降低線上人員可能因為生產觀念欠缺而造成不必要的成本浪費。因為半導體的製程至少都有200~300道以上，而且每種產品的途程跑法又不盡相同，限制又相當多，所以就會造成產線經常性的某個點堆貨，但是機台卻又可能沒在RUN，這牽扯到從投片開始就要計算每種機台針對產品造成的限制，以及產能足不足夠？在現場的排貨方式可能需要自行 Study 一個符合目標需求的 Scheduling 或 Dispatching，減少機台Idle 時間或是讓急件能夠優先 Push，所以對每種機台的特性都必須要有一定程度的了解比方某機台一次只能上2批、某某機台卻可上6批，Recipe 的 Processing Time 有多久？某些 4~5 小時、某些只要幾分鐘，如果了解度不夠就會沒有掌控力，也無法下達指令。每日的生產會議就是在 Review 昨日的各項生產指標，像是產出啦、急件的 C/T 啦，如果做不到就要說明原因，並且提出改善的方針，一切都要合乎邏輯，並且要能掌握住大量的生產資訊而且數量化。比方昨日某某重點機台從幾點到幾點當機啦，產線課長在這一塊做了那些應變措施？產品怎麼安排？RUN 貨優先度為何？有沒有請設備優先處理？又好比說昨日為何 Movement 不好？Turn Rate 的數據有問題？可能就要說明是因為某區域某機台人力不足，該機台的有效產出不佳，而對 Movement 影響有多大？(數據)因此昨日有針對此部分即時做人力調度，相對被安排調度的區域則是怎麼分派工作等等都要說明清楚。諸如此類的事件，想要從大方向到很細微的點都能夠說明，其實是相當困難的，頭腦不夠清楚的人、對生產沒Sense的人、經驗不足的人，被釘在牆上動彈不得是很常見的...另外產線上的 WIP 推移也要能預測未來的動向，提早做出應對，比方某某製造途程的產品數量很大，在未來2~3天會經過某個將因為產能不足而預計造成瓶頸的機台，那該機台是否都準備好了？是不是該和設備 Check 要提早還是延後 PM？該有的 Recipe 都有嗎？是不是需要製程先去協助建 Recipe？實際上生產時要針對瓶頸加以管理，像是限制理論的應用，瓶頸前的工作要趕工、要機台調配，瓶頸站的生產批量要大，要減少 Setup 的產能浪費、Idle 時間的產能浪費，這方面都要隨時監控並做調派。在生產會議上則要針對該機台提出相關報告，像是 Dispatching Rule 為何？預計日產出多少？未達到 Target 的原因？更進一步來說，機台為何待料？為何實際的 Up Time 值未達標準？這就會牽扯到機台或產品限制條件、人員效率、產品上貨的批數問題、送貨的 Rule 為何、甚至和控檔片的管理機制都有關係，這方面到底做過哪些 Action都必須在必要時要加以說明。以上此類零零總總的瑣碎事情很常見，在旺季產線量大時這些問題更是多到不行，影響範圍又很廣，所以對產線真的要有一定的敏銳度，不然很容易 Loss 掉，然後開會被 K---&gt; 壓力大 ---&gt; 逆境中成長 or 殞落...XD2. 即時處理產線意外事件，如：工安異常、小姐缺失、急件、線上的系統使用問題...待產線有時候最怕的就是突發的事件，如果一整天都沒發生那真的算是運氣很不錯，有些是屬於工業安全的部分有異常，比如 FAB 內有發生異味啦、地震啦、管線漏出不明液體...諸如此類可能影響生產的部分你都要管就對了，要懂得如何處理事情很重要，因為發生什麼事都有可能，遇到沒遇過的事一點都不奇怪。有時候小姐反應有異味，進去聞了老半天也感覺不到，這真的是每個人感受不同，但小姐反應事情又不得不處理，只好到處問工程師、廠務，到處查原因。發生地震的話，又要趕緊掌握每一個生產區域的機台狀況、產品狀況，然後要回報一堆有的沒的，小姐遇到不會處理的事又會一直 Call 過來，常常一片混亂，尤其是上頭又喜歡在這時候叫你做這做那的。另外也很怕小姐發生缺失事件，一發生就要查明原因，因為有些小姐會不誠實不然就是連自己怎麼發生都說不清楚，然後不外乎是看監視器錄影，現場還要拍照推斷原因，做8D Report準備向上頭報告缺失發生的真因和改進方式，偶爾還會有小姐擺臭臉給你看，然後還要約談留紀錄並寫傳簽單、重新教育正確觀念，有時一忙起來其他事都不用做了。3. 處理線上小姐的各種事情：休假問題、教育訓練問題、考核、人員調度評估，　 以及三不五時會打電話來問的一堆奇奇怪怪的事，有的真的會覺得是來亂的...0rz就像很多人所說，牽扯到人的問題才是最麻煩的，旺季的時候要加班，淡季的時候要休假，要嘛是加班人數有問題、排不定，要嘛就是休假又一堆小姐有意見，每個人需求都不同，人都是自私的，勾心鬥角有的沒的，這時候產線課長/主任就要扮演協調的角色，一張嘴要能說得別人信服，中間眉眉角角的事情很多，當然，也絕對不會所有人都能配合、都能服你，記恨啦～鑽漏洞啦～背後捅你一刀的事都會有可能發生。人員的教育訓練需要評估人選，因為又會牽扯到機台調度上的需求，有些人身高夠、有些人細心度好、有些人動作快，最適宜的就是根據每個人特性去做分派，常常需要去計算人力、做 List 和上頭說明，教育訓練也需要有老的帶菜的，那又會造成產線上實際 RUN 貨人力受到影響，產出就會變差，總之問題很多都要想辦法解決...和線上的幹部必須要保持良好的關係，因為有很多 Detail 的事必須要交給他們去做，他們對產線上一些細節一定比我們清楚，某些人表現的優或劣，常常也要透過他們去知曉，當然有些幹部執行力強或觀念ok、有些幹部就相對弱，這些主任/課長都必須要掌握好，適時和她們溝通或灌輸正確的觀念。像小姐考核的加分扣分、出席率、個人品質，必要時也需要約談小姐和她們說明以及提醒，有時候也會遇到小姐 Call 來講說產品或是系統異常，沒辦法 Run 貨，這時候又需要去查明是哪方面的問題，因為有的小姐表達的不清楚，是系統本身問題還是小姐的操作問題？相對的就必須要對機台的操作和各種系統的使用有一定程度的了解，不然就沒辦法解決問題。總之，人員管理的部分雜事是最多的，也最常需要和底下溝通協調。4. 工程改善專案的制定，相對要有一定程度的生管基礎，不外乎是讓產線運作更加順利，降低一些有形或是無形的成本。除了平常要做的如之前所述的雜事之外，有時也會被要求要做 Study、要撈資料作分析，這些專案的部分就會變得只好在下班或休假時間來做，雖然很賽但是相對來說也會對產線更加深入了解，當然，這些都是要有成果報告的。專案的部分從小到大都有，比方線上環境的6S、Layout(EX:治工具機台擺放、移動動線)、6-Sigma 專案執行、控檔片流程管理、表單精簡改進(EX:電子化).....另外還有屬於比較具 IE 專業性的，像是需要工作研究或是精實生產概念等等之類延伸的專案，這類型就比較花腦力，是非強制性屬於個人提出改善的部分。----------------------------------------------------------------------------以上這些都是製造課長/產線主任要做的事，也許還有很多沒說到，但大致上就是這樣啦，現在有學弟來的時候，真的都不知道他們能撐多久，畢竟已經看過不少幾個月就陣亡的新人了，雖然靠近產線端相對辛苦，但還是有不少有趣的地方，也能夠學習到很多實務上的運作情況。如果是剛畢業要投履歷的新鮮人，可以參考看看，當每家公司的情況都不一樣，不過累是一定的，至於
